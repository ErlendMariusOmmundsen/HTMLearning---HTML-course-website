Group 1 - requirements draft

by Martin Skurtveit, Joakim Fremstad, Erlend Marius Ommundsen, Lærke Svendsen
Written 20.09.2018

Choice of client

For this project, we have chosen an imaginary client who wishes to create a website for the HTML-courses, and other courses, they offer here in Trondheim. The company HTMLearning, aims to promote and sell their courses through a website. They have requested a website with user registration and a plain design to make it easier for inexperienced users to navigate the site.

Purpose and Goals

The site shall inform users about upcoming courses, prices, and other details about the courses offered, in addition to making it possible to register to a course. In this way the company wishes to reach more of their user base, and make it easier to use their services. To goal is to increase the number of participants in the courses.
As a second goal the company wishes to become more known, and improve their reputation. This is achieved by making the site easy to navigate, always available across devices.

Audience

The websites intended audience is people that want to learn HTML and additional web technologies, but mainly students and people who need it for work. They can be completely new to coding, or they can already have some experience. “HTMLearning” should cater to every level of coding skill. Since the user has different starting points when it comes to knowledge, they need detailed information about the courses that the site provides. One challenge with this audience is the wide variety of devices they use. This could prove challenging when trying to implement a consistent design.

Content of site

The company wants to have a front page, with a short introduction to the website and HTML itself. They also want a menu to navigate to the rest of the site’s content, accessible from all parts of the website. From the menu bar, a user should be able to reach the most essentials part of the site: Front page, Courses overview, About the company, FAQ and Log in. The courses overview should be well-organized, to let the user easily separate the different types of courses. There should also be a part of the website that shows how to come in touch with the company.

Requirements

Functional requirements

  1. (Very High priority)Navigation menu in the top of the page with links to: Front page, About us page, Courses overview page, Log in, FAQ.
  2. (High Priority)The web page should contain an overview of all course types.
    1. From here, the user should only be able to see essential info about pricing, course length, time and date etc.
    2. Each course type should link to a page with more detailed descriptions of the coming courses of this type.
    3. Detailed course description should include a small window with the course location in Google Maps.
  3. (Medium-High)Frequently Asked Questions site
    1. (Low priority)FAQ questions should only display the question, then when it is clicked the answer will be displayed under the question.
  4. (Medium-High priority) Contacting information, preferably in the footnote.
  5. (High priority) Users should be able to register for courses.
    1. The registration form should output a success message after “registering”.
  6. (Medium priority) The navigation menu should contain a login button that takes the user to a (fake) profile page.
  7. (Low priority) The user should be able see the course ratings and/or reviews and rate and/or review them themselves.
  8. (Low priority) Under CSS courses: Demonstrate how a webpage looks without CSS.
  9. (Medium priority) All pages should have a plus/minus to increase/decrease text size. (Refer to Non-functional req.3)

Non-functional requirement

  1. Every page should load in less than 1 second.
  2. The user should be able to navigate the menu from all pages.
  3. All text should be readable.
  4. The web page should be easily usable on Android and IOS.
  5. The page should be easily maintainable.
  6. The page should comply with the GDPR regulations.
  7. Availability: The page should update after one pushes to Github.
