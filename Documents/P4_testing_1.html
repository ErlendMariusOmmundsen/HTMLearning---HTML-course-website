Testing of the website

The test subject will be asked to perform the following list of tasks on our website. We will take notes on how the user manages to do the tasks. After the test has concluded, the test subject may give some comments about his/her user experience.

  1. View the front page and find the main content
  2. Create a user
  3. Login with this user
  4. Find a course that looks interesting
  5. Find out where and when the course takes place
  6. Find the contact information of the company
  7. Look up who works for the company

Tester Audience

As we ourselves are students, the student part of our audience is much easier to reach, than the working part. Each of us were required to interview at least one person.

Testing platforms

Feedback summary

Design

Some conventions, like the contact information in the footer, are good to have.

User became confused about the "Register for course" button, as it was believed that registering referred to signing in, and the user therefore had to sign in before registering for the course.

One user thought that the alerts when registering for course were annoying. User rather wanted a banner to popup under the navbar instead.

Other users mentioned that some pages had almost too much information on them. They said that the course pages were a bit cluttered.

Another user pointed out that the site looks good and does its task well. He pointed out that he thinks the course page looks a bit barren on a wide screen, and suggested that we limit the width. He also had some trouble understanding the process of signing up, because of the validating input.

Navigation

Easy to get around. On a lot of the tasks the users eyes automatically moved to the navbar.

When looking for the next course date, the user immediately went to the specific course page, and did not notice the text on the link.

Responsive design

Still a bit lacking, as some elements overlap on smaller screens. User noted that the contact information in the footer was only partly visible, but this was fixed after the test. A user thought the mobile version was working alright, and had no problems using it even though we have prioritized PC browsing.

Our response to the feedback

Our test subjects have given us some useful thoughts and suggestions on how to improve the site. Most of the feedback is indeed actionable, for instance we could limit the width on the course page, and fix some of the overlap. We could also add more detailed messages when validating input in the login and signup forms.

Reflection

We feel that the project experience as a whole is positive. We got a feel of the limits and possibilities for the different languages, and we learned to work as a team where everyone had different strengths. We also used this project to learn how to use Git at the same time, which has been rewarding. However it takes a lot of time and work for only a part of the total grade. Although this is a basic course, to learn the basic functions of HTML, CSS, and JS. We feel it is very limiting, because there are very many useful and important parts that are neglected, such as flexboxes.
